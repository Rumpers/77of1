<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| creator voice note → Telegram → Hermes | untrusted file content; size + duration validated before upload |
| Hermes → Replit Object Storage | trusted (managed PaaS); bucket private |
| /revoke_voice → consent_grants update | trusted (creator authenticated via telegram_id mapping) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08-01 | Tampering | malicious file uploaded via Telegram voice | mitigate | Telegram pre-validates file type as voice (ogg/mp3); duration check ≥6s; size implicit limit via Telegram (max ~50MB per file) — no further validation in Phase 2 |
| T-02-08-02 | Info Disclosure | voice reference URL leak | mitigate | Private bucket; signed URLs (TTL); voiceReferenceUrl stored server-side only; never exposed to fan SPA |
| T-02-08-03 | Repudiation | creator denies giving voice consent | mitigate | consent_grants row with version + ipHash + granted_at remains; revokedAt set on revocation, NEVER deleted (immutable audit) |
| T-02-08-04 | Tampering | revoke_voice replay / race with new voice consent | accept | Idempotency: re-running /revoke_voice on already-revoked grant is a no-op (worker selects WHERE granted=true); race window is small (seconds) and N=1 |
| T-02-08-05 | DoS | unbounded uploads (creator spams /voice) | accept | Telegram rate-limits per user; N=1 means founder can manually intervene; bucket has implicit Replit quotas |
| T-02-08-06 | Compliance violation | voice not deleted within 60s SLA | mitigate | Worker logs warn at >60s; manual retry path via founder; Phase 4 adds metric alert |
| T-02-08-07 | Tampering | voice_reference.wav overwrite race | accept | Single key per creator; overwrite is the intent (creator records new sample) |
| T-02-08-SC | Tampering | @replit/object-storage SDK install | mitigate | Task 0 founder approval; npm view check before install; raw-fetch fallback if SDK is [SUS] |
</threat_model>
