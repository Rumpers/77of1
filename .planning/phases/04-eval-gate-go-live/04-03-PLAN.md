<objective>
Build the go-live gate as a founder-authed admin endpoint: POST /api/admin/twin/:creatorId/activate
reads the latest eval run via isGoLiveEligible and only flips twins.status to "active" when the
hard-limit and injection categories both passed 100%. This is the vertical slice that makes
"the twin cannot be set live unless the eval gate passes" enforceable.

Purpose: EVAL-01 success criterion 1 — the twin cannot go live unless hard-limit and injection
scores are both 100%. The existing twins.status="active" check in routes/twin.ts is the runtime
gate; this endpoint is the controlled one-time transition that respects the eval verdict.

Output: A founder-auth middleware, the activate route wired into the api-server router, and tests
proving the 401 / 422 / 200 paths.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet → admin activate endpoint | A new mutating admin route that flips a twin live. Must be unreachable by fans/creators. |
| client-supplied :creatorId → DB update | A path param used in a parameterized update; must be a valid UUID. |
| eval verdict → status transition | The activate decision depends on eval_runs; a stale/forged verdict must not enable go-live. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Elevation of privilege | POST /admin/twin/:creatorId/activate | mitigate | founderAuth middleware requires ADMIN_API_TOKEN bearer (constant-time compare); fails closed when the secret is unset. Not reachable by fan/creator sessions (no cookie/session path grants it). |
| T-04-03-02 | Spoofing | weak/missing admin secret | mitigate | timingSafeEqual comparison defeats timing oracles; missing ADMIN_API_TOKEN → 401 for everyone (fail closed). |
| T-04-03-03 | Information disclosure | 422 error body | mitigate | The eval_gate_failed response states the policy generically and does NOT enumerate which eval cases failed (ASVS V13). |
| T-04-03-04 | Tampering | forged go-live via direct status write | accept | DB-direct tampering is an N=1 operational risk; the activate route + eval_runs row is the audit trail (04-RESEARCH Security Domain). Founder is the only operator. |
| T-04-03-05 | Input validation | :creatorId path param | mitigate | Handler validates :creatorId as a UUID (400 otherwise); Drizzle parameterizes the update (ASVS V5). |
</threat_model>
