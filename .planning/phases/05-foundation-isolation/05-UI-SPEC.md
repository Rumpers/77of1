## CSS Isolation Contract

### Hard Rule: Zero Leakage Guarantee

The fan page is dark-mode-only. Its tokens are defined in the `.dark` block of
`index.css` and consumed via `--primary`, `--background`, `--foreground`, etc. The
`body` element applies `bg-background text-foreground` globally. Marketing tokens
MUST never appear on a `:root` or `body` selector — they would contaminate the fan
page.

**The isolation mechanism is a mandatory prerequisite for all Phase 6 component work.**

### Mechanism

Append one new block to `artifacts/web/src/index.css`, AFTER the existing
`@layer utilities` block:

```css
@layer marketing-tokens {
  [data-surface="marketing"] {
    /* PLACEHOLDER VALUES — replace wholesale via UI-SPEC before Phase 6 */
    --mkt-bg:            #ffffff;
    --mkt-fg:            #111111;
    --mkt-surface-1:     #f5f5f5;
    --mkt-surface-2:     #ebebeb;
    --mkt-border:        #e0e0e0;
    --mkt-accent:        hsl(263 80% 58%);
    --mkt-accent-fg:     #ffffff;
    --mkt-accent-hover:  hsl(263 80% 50%);
    --mkt-muted-fg:      #666666;
    --mkt-radius-sm:     0.375rem;
    --mkt-radius-md:     0.625rem;
    --mkt-radius-lg:     1rem;
    --mkt-font-sans:     'Inter Variable', 'Noto Sans JP Variable', system-ui, sans-serif;
    --mkt-text-xs:       0.75rem;
    --mkt-text-sm:       0.875rem;
    --mkt-text-base:     1rem;
    --mkt-text-lg:       1.125rem;
    --mkt-text-xl:       1.25rem;
    --mkt-text-2xl:      1.5rem;
    --mkt-text-3xl:      1.875rem;
    --mkt-text-hero:     clamp(2.25rem, 5vw, 4rem);
    --mkt-text-section:  clamp(1.5rem, 3vw, 2.25rem);
    --mkt-leading-tight: 1.2;
    --mkt-leading-body:  1.6;
    --mkt-leading-cjk:   1.8;
    --mkt-spacing-xs:    0.25rem;   /* 4px */
    --mkt-spacing-sm:    0.5rem;    /* 8px */
    --mkt-spacing-md:    1rem;      /* 16px */
    --mkt-spacing-lg:    1.5rem;    /* 24px */
    --mkt-spacing-xl:    2rem;      /* 32px */
    --mkt-spacing-2xl:   3rem;      /* 48px */
    --mkt-spacing-3xl:   4rem;      /* 64px */
  }
}
```

**Why `@layer marketing-tokens` and not a standalone CSS block:**
Tailwind v4 uses native `@layer`. Any rule defined OUTSIDE a layer has higher
implicit specificity than all layered rules, including Tailwind utilities. Wrapping
the marketing token block inside a named layer ensures it cannot accidentally override
fan-page utilities regardless of selector specificity.

**Why `[data-surface="marketing"]` and not a class:**
Attribute selectors cannot appear in class composition shortcuts. They provide a clear
semantic boundary: the marketing surface is a structural DOM contract, not a style
convenience. The root element of `home.tsx` must render with
`data-surface="marketing"` on its outermost div.

### Token Category Inventory

The following categories are defined in Phase 5 with placeholder values.
Phase 6's UI-SPEC will replace all placeholder values with final brand decisions.

| Category | Tokens | Purpose |
|----------|--------|---------|
| Surface | `--mkt-bg`, `--mkt-surface-1`, `--mkt-surface-2` | Background hierarchy (page, section, card) |
| Text | `--mkt-fg`, `--mkt-muted-fg`, `--mkt-accent-fg` | Foreground hierarchy |
| Accent | `--mkt-accent`, `--mkt-accent-hover`, `--mkt-accent-fg` | CTA button, links, interactive highlights |
| Border | `--mkt-border` | Dividers, card borders |
| Radius | `--mkt-radius-sm`, `--mkt-radius-md`, `--mkt-radius-lg` | Border radius scale |
| Typography | `--mkt-font-sans` | Font family stack |
| Type scale | `--mkt-text-xs` through `--mkt-text-hero`, `--mkt-text-section` | 8 size steps + 2 fluid clamp values |
| Line height | `--mkt-leading-tight`, `--mkt-leading-body`, `--mkt-leading-cjk` | 3 presets: headings / body / CJK body |
| Spacing | `--mkt-spacing-xs` through `--mkt-spacing-3xl` | 7-step 4px-base scale |

### Enforcement Rules for Phase 6+

All marketing components MUST:
- Only consume `--mkt-*` tokens
- Never reference `--primary`, `--background`, `--foreground`, `--card`, or any
  fan-page token by name
- Be contained within a DOM subtree where `[data-surface="marketing"]` is an ancestor
- Never write rules outside `@layer marketing-tokens` that target element or universal
  selectors (no `*, body, :root` rule additions)

### Verification Contract

After every marketing CSS commit, a visual regression check is required:
visit `/en/claire` (or any valid fan handle) and confirm the fan chat page renders
with the correct dark background (`#0f0f0f`) and brand violet primary. Any style bleed
means a token has escaped the `[data-surface="marketing"]` scope.

## Spacing Scale

Uses the `--mkt-spacing-*` token series defined in the CSS isolation block above.
All values are multiples of 4px.

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--mkt-spacing-xs` | 4px | Icon gaps, inline padding, tight badge padding |
| sm | `--mkt-spacing-sm` | 8px | Compact element spacing, icon-to-label gap |
| md | `--mkt-spacing-md` | 16px | Default element spacing, card padding (mobile) |
| lg | `--mkt-spacing-lg` | 24px | Card padding (desktop), nav item spacing |
| xl | `--mkt-spacing-xl` | 32px | Section inner padding, layout column gaps |
| 2xl | `--mkt-spacing-2xl` | 48px | Major section vertical breaks |
| 3xl | `--mkt-spacing-3xl` | 64px | Page-level section vertical spacing |

Exceptions:
- Touch targets (nav items, CTA buttons): minimum 44px height per WCAG 2.5.8
  (AA requirement for touch targets). Achieves this via `min-height: 44px` and
  appropriate vertical padding — not a deviation from the spacing scale, but an
  override for interactive element constraints.

## Typography

### Font Loading Contract (MKT-17 — Non-Negotiable)

This section is a real visual contract for Phase 5, not a deferral.
It governs what the user sees during font loading on slow connections.

**Packages to install (Phase 5 task):**
```
pnpm --filter @workspace/web add @fontsource-variable/inter @fontsource-variable/noto-sans-jp
```

**Remove from `artifacts/web/index.html`:**
```html
<!-- DELETE THESE THREE LINES -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap" rel="stylesheet">
```

**Add to `artifacts/web/src/main.tsx` (or a dedicated font import file):**
```typescript
import '@fontsource-variable/inter';
import '@fontsource-variable/noto-sans-jp';
```

Fontsource variable packages ship `font-display: swap` in their bundled `@font-face`
declarations. No manual override is needed for swap behavior.

**Add to `artifacts/web/index.html` `<head>` for the Noto Sans JP 400-weight woff2 preload:**
```html
<link rel="preload" as="font" type="font/woff2"
  href="/node_modules/@fontsource-variable/noto-sans-jp/files/noto-sans-jp-latin-wght-normal.woff2"
  crossorigin="anonymous" />
```

NOTE: Fontsource places files in `node_modules` at build time; Vite resolves these
paths. Confirm the exact woff2 path after install by inspecting
`node_modules/@fontsource-variable/noto-sans-jp/files/`. If the file path differs
from the above, correct the preload href accordingly.

**Fallback font stack (applied via `--mkt-font-sans`):**
```
'Inter Variable', 'Noto Sans JP Variable',
system-ui, -apple-system, "Hiragino Sans", "Meiryo", sans-serif
```
- `Hiragino Sans` — Japanese system font on macOS/iOS (excellent CJK coverage)
- `Meiryo` — Japanese system font on Windows
- These fallbacks appear immediately when web fonts are loading (swap behavior),
  preventing invisible text (FOIT) on slow connections

**Visual acceptance behavior (definition of done for MKT-17):**
In Chrome DevTools with Network throttled to "Slow 3G":
- Hero text (any locale) shows a system fallback font IMMEDIATELY on first paint
- No period of invisible/blank text occurs at any point during loading
- When `@fontsource-variable/noto-sans-jp` finishes loading, a font swap occurs —
  this visual shift is acceptable; invisible text is not

### Type Scale

PLACEHOLDER VALUES — Phase 6 UI-SPEC will replace sizes with final brand decisions.
The TOKEN NAMES are locked; the values are not.

| Role | Token | Placeholder Value | Weight | Line Height |
|------|-------|------------------|--------|-------------|
| Body | `--mkt-text-base` | 16px (1rem) | 400 | `--mkt-leading-body` (1.6) |
| Label / small | `--mkt-text-sm` | 14px (0.875rem) | 400 | `--mkt-leading-body` (1.6) |
| Subheading | `--mkt-text-xl` | 20px (1.25rem) | 600 | `--mkt-leading-tight` (1.2) |
| Section heading | `--mkt-text-section` | clamp(1.5rem, 3vw, 2.25rem) | 700 | `--mkt-leading-tight` (1.2) |
| Hero display | `--mkt-text-hero` | clamp(2.25rem, 5vw, 4rem) | 700 | `--mkt-leading-tight` (1.2) |

**CJK-specific overrides (non-negotiable; applies in all phases):**
- All CJK text uses `line-height: var(--mkt-leading-cjk)` (1.8) — not the Latin body
  line height (1.6). CJK characters are square and feel cramped at Latin rhythm.
- Apply `word-break: auto-phrase` to all headings that may contain Japanese text.
  Supported in Chrome 119+ (covers the primary JP/TW mobile audience).
- Apply `line-break: strict` to Japanese body paragraphs to prevent breaks before
  Japanese punctuation marks.
- Apply `overflow-wrap: break-word` to all marketing text containers.
- Never use `white-space: nowrap` on a container that may receive CJK text.

## Color

PLACEHOLDER VALUES — All hex/hsl values below are neutral placeholders that satisfy
the structural contract only. Phase 6 UI-SPEC will replace every value with the final
brand palette decisions (light vs. dark surface direction, accent hue, etc.).

The STRUCTURE (token names + roles) is locked. The VALUES are not.

| Role | Token | Placeholder Value | Usage |
|------|-------|------------------|-------|
| Dominant surface (60%) | `--mkt-bg` | `#ffffff` | Page background under `[data-surface="marketing"]` |
| Secondary surface (30%) | `--mkt-surface-1` / `--mkt-surface-2` | `#f5f5f5` / `#ebebeb` | Cards, section alternates, elevated areas |
| Accent (10%) | `--mkt-accent` | `hsl(263 80% 58%)` | Reserved for: primary CTA button background, active nav indicator, focus ring |
| Accent hover | `--mkt-accent-hover` | `hsl(263 80% 50%)` | CTA button hover state only |
| Accent foreground | `--mkt-accent-fg` | `#ffffff` | Text/icon on accent backgrounds only |
| Body text | `--mkt-fg` | `#111111` | Primary text on `--mkt-bg` surfaces |
| Muted text | `--mkt-muted-fg` | `#666666` | Supporting copy, captions, meta text |
| Border | `--mkt-border` | `#e0e0e0` | Card borders, section dividers |

Accent reserved for: primary CTA button (background), active nav indicator,
focus ring on interactive elements. Not for decorative separators, body text
emphasis, or icon fills.

No destructive actions exist in Phase 5 (plumbing only). Phase 6 adds no destructive
actions on the marketing surface. If a destructive action is ever added to the
marketing surface, define a separate `--mkt-destructive` token at that time — never
borrow the fan page's `--destructive`.

**Isolation invariant:** The `.dark` class in `index.css` MUST NOT affect
`[data-surface="marketing"]` children. The marketing surface is explicitly not
a dark-mode surface. If the `body` receives the `.dark` class, `@layer marketing-tokens`
scoped under `[data-surface="marketing"]` will override the dark tokens within that
subtree — this is the correct and intended behavior.

## Route Safety Contract (MKT-20)

Phase 5 adds no new named routes to `App.tsx`. The existing route order is:

```
/              → redirect to /en
/:locale/onboard → (and sub-routes) — ABOVE the fan catch-all
/:locale/account/data-request → DsarPortal — ABOVE the fan catch-all
/:locale/dashboard → CreatorDashboard — ABOVE the fan catch-all
/:locale       → MarketingPage (replaces the 8-line HomePage placeholder)
/:locale/:handle → FanPage (fan catch-all — MUST remain last)
/payment/*     → payment pages
```

Phase 6 will add marketing named sub-routes. When added, EVERY marketing named
sub-route must be placed ABOVE `/:locale/:handle` in the `Switch`. No exceptions.
Failure mode: a visitor to `/en/[section-name]` renders the FanPage 404 state and
fires an API call to `/api/twin/[section-name]`.

## Code-Split Contract (Pitfall 11 prevention)

Phase 5 adds `React.lazy()` wrapping to ALL page components in `App.tsx`. This is a
prerequisite for safe marketing site addition — without it, fans downloading the fan
chat page also download the entire marketing site code.

```typescript
// App.tsx — ALL page imports converted to React.lazy
const MarketingPage = React.lazy(() => import('./pages/home'));
const FanPage = React.lazy(() => import('./pages/fan-page'));
const DsarPortal = React.lazy(() => import('./pages/dsar-portal'));
// ... all other pages
```

Each lazy-loaded page must be wrapped in a `<Suspense fallback={null}>` (or a minimal
spinner) in the route render function.
