# Phase 6 — UI Design Contract: Marketing Components & Navigation

> Visual and interaction contract for the Phase 6 marketing component build.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope:** `artifacts/web/src/components/marketing/` and `artifacts/web/src/pages/home.tsx` only.
> Fan chat page (`02-UI-SPEC.md` governed) is out of scope and must remain visually unchanged.

> **Authority:** DESIGN.md is the locked aesthetic source of truth. This spec translates its
> decisions into per-component contracts. Any deviation requires explicit user approval and

`[data-surface="marketing"]` subtree.

## Spacing Scale

Source: DESIGN.md spacing section + `index.css @layer marketing-tokens` verified values.

All marketing spacing references `--mkt-spacing-*` tokens via Tailwind arbitrary-value syntax.
The base unit is 4px. Standard 8-point scale applies.

| Token | CSS Var | Value | Usage in Phase 6 |
|-------|---------|-------|------------------|
| xs | `--mkt-spacing-xs` | 4px (0.25rem) | Icon gaps, badge inner padding, inline gaps |
| sm | `--mkt-spacing-sm` | 8px (0.5rem) | Compact element spacing, pill button vertical padding |
| md | `--mkt-spacing-md` | 16px (1rem) | Default element spacing, card inner padding (horizontal) |
| lg | `--mkt-spacing-lg` | 24px (1.5rem) | Section gutters (horizontal), card inner padding (vertical) |
| xl | `--mkt-spacing-xl` | 32px (2rem) | Layout gaps between grid items |
| 2xl | `--mkt-spacing-2xl` | 48px (3rem) | Sub-section vertical padding |
| 3xl | `--mkt-spacing-3xl` | 64px (4rem) | Section-level vertical padding (mobile fallback) |

**Section vertical padding (desktop):** ~84px top/bottom for content sections; hero ~90–120px.
These are not exact multiples of the 8pt scale and are expressed as one-off Tailwind values
(`py-20` = 80px, `py-24` = 96px) — acceptable for section-level layout rhythms.

**Nav height:** 56px (3.5rem) — matches `h-14` Tailwind class.

**Max content width:** 1120px via `max-w-[1120px]`. Gutters: 24px (`px-6`) on all sections.

**Mobile overflow invariant (MKT-11):** Every section wrapper applies `w-full max-w-[1120px] mx-auto px-6`.
The `[data-surface="marketing"]` root div sets `overflow-x: hidden`. All absolutely-positioned
decorative elements (HeroOrb, bloom meshes) must have `overflow-hidden` on their containing parent.
Grid children use `min-w-0` to prevent flex/grid blowout. After each section is built, verify:
`document.documentElement.scrollWidth === 375` at 375px viewport.

**Touch targets:** CTA buttons minimum 44px tall. Nav locale pill buttons minimum 32px tall (acceptable
for desktop-first nav; mobile locale switching is via the footer duplicate).

Exceptions: none beyond the section-level padding described above.

## Typography

Source: DESIGN.md typography section + `index.css @layer marketing-tokens` verified scale.

**Font application rule:** In marketing components, always use `style={{ fontFamily: "var(--mkt-font-display)" }}`
or `style={{ fontFamily: "var(--mkt-font-sans)" }}` rather than the `font-sans` Tailwind utility class,
because `font-sans` maps to `--app-font-sans` (the fan-page Inter stack), not `--mkt-font-sans` (Geist).

### Phase 6 Declared Sizes (4 only)

| # | Role | Size | Weight | Font | Line Height | Usage |
|---|------|------|--------|------|-------------|-------|
| 1 | **Display** | `clamp(2.25rem, 5vw, 4rem)` | 700–800 | Bricolage Grotesque | 1.04 | `<h1>` hero headline AND `<h2>` section headings — same clamp, rendered size differs by viewport |
| 2 | **Subheading** | `1.5rem` (24px) | 600 (label) / 800 (numeral) | Bricolage Grotesque (numeral) / Geist (label) | 1.2 | Card titles `<h3>`, step label `<h3>`, step numeral — weight + font family differentiate; size is shared |
| 3 | **Body** | `1rem` (16px) | 400 | Geist | 1.6 | Section subtitles, subheadlines, card descriptions, footer copy, hero subheadline |
| 4 | **Small** | `0.875rem` (14px) | 400 | Geist | 1.6 | Section eyebrow labels, footer legal line, "coming soon" badge label, nav locale pills, twin attribution line |

**Differentiation within Subheading (1.5rem):**
- Step numerals ("01" / "02" / "03"): `1.5rem`, weight **800**, `var(--mkt-font-display)` (Bricolage Grotesque), color `--mkt-accent`
- Step labels `<h3>` and card titles `<h3>`: `1.5rem`, weight **600**, `var(--mkt-font-sans)` (Geist), color `--mkt-fg`

**Differentiation within Small (0.875rem):**
- Eyebrow labels: `0.875rem`, weight 500, uppercase, `letter-spacing: 0.14em`, color `--mkt-accent`
- Footer legal / badge / nav pills / attribution: `0.875rem`, weight 400, no transform

**Available but unused in Phase 6 (`--mkt-text-*` tokens that exist in the token map but are NOT
declared as Phase 6 in-use sizes):**
`--mkt-text-xs` (0.75rem), `--mkt-text-lg` (1.125rem), `--mkt-text-xl` (1.25rem),
`--mkt-text-3xl` (1.875rem) — these remain in CSS for completeness but no Phase 6 component
renders at these sizes. If a future executor is tempted to use them, a new checker run is required first.

**Declared weights:** 400 (regular) and 600 (semibold) for Geist body/UI. 700–800 for Bricolage
Grotesque display. This satisfies the 2-weight body constraint; display font has its own weight range.

**CJK leading:** `--mkt-leading-cjk: 1.8` — mandatory for Japanese and Traditional Chinese legibility.
Apply `lang="ja"` / `lang="zh-TW"` at the `[data-surface="marketing"]` root (or section wrapper) when
rendering JA/ZH-TW locale. The `word-break: keep-all` rule must be set for ZH-TW; `overflow-wrap: break-word`
for both CJK locales.

**Display heading tracking:** `-0.02em` on `<h1>` and `<h2>`.
**Eyebrow tracking:** `+0.14em` uppercase (`text-transform: uppercase; letter-spacing: 0.14em`).

## Color

Source: DESIGN.md color section + `index.css @layer marketing-tokens` verified hex values.

**Palette approach:** Dark-first, violet-ink canvas. No light variant for v1. The 60/30/10 split is
applied within the `[data-surface="marketing"]` scope only.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) canvas | `--mkt-bg` | `#0D0A14` | Page background, `[data-surface="marketing"]` root fill |
| Secondary (30%) surfaces | `--mkt-surface-1` / `--mkt-surface-2` | `#16111F` / `#1F1830` | Cards (pillar, demo, step), nav background (`/90` opacity), footer |
| Accent (10%) brand violet | `--mkt-accent` | `#7C3AED` | Reserved — see accent list below |
| Accent hover | `--mkt-accent-hover` | `#6D28D9` | CtaButton hover state only |
| Accent foreground | `--mkt-accent-fg` | `#FFFFFF` | Text on accent fills (CTA button labels, active locale pill) |
| Primary text | `--mkt-fg` | `#F4F1FA` | Headings, body text, wordmark |
| Secondary text | `--mkt-muted-fg` | `#9D94B5` | Section subtitles, card descriptions, footer secondary copy |
| Border / divider | `--mkt-border` | `#2A2238` | Card borders, nav bottom border, section dividers |
| Glow from | `--mkt-glow-from` | `#7C3AED` | HeroOrb radial-gradient start, gradient headline spans, CTA shadow |
| Glow to | `--mkt-glow-to` | `#D946EF` | HeroOrb radial-gradient end, footer mesh bloom |
| Destructive | not applicable | — | No destructive actions in this phase |

**Accent reserved for (explicit list):**
1. `CtaButton` primary variant fill (`bg-[--mkt-accent]`)
2. Active locale pill in `MarketingLocaleSwitcher` (`bg-[--mkt-accent]`)
3. Hero headline gradient span (`background: linear-gradient(to right, var(--mkt-glow-from), var(--mkt-glow-to))` + `background-clip: text`)
4. `HeroOrb` radial-gradient bloom
5. CTA button glow box-shadow (`color-mix(in oklch, var(--mkt-glow-from) 40%, transparent)`)
6. Section eyebrow text (accent color, not fill)
7. Step numerals in HowItWorksSection (accent color, weight 800, display font)

Accent is NOT used for: body text, nav wordmark, card borders, icon fills (icons use `--mkt-muted-fg`
at rest; `--mkt-fg` on hover).

**Contrast compliance:**
- `--mkt-fg` (`#F4F1FA`) on `--mkt-bg` (`#0D0A14`): ~16:1 — AAA
- `--mkt-muted-fg` (`#9D94B5`) on `--mkt-bg`: ~6.5:1 — AA
- `--mkt-accent-fg` (`#FFFFFF`) on `--mkt-accent` (`#7C3AED`): ~4.6:1 — AA (buttons)

### 3. CtaButton (MKT-08, MKT-09)

**Single reusable primitive.** Rendered in exactly 4 positions: MarketingNav, HeroSection, CtaSection
(mid-page), MarketingFooter. All instances use identical wording from i18n keys.

**CTA label (primary, all positions):** `t.hero.cta_primary` / `t.cta.button` — EN: **"Start with Telegram"**
**Fallback label:** `t.hero.cta_no_telegram` / `t.cta.no_telegram` — EN: **"No Telegram? Contact us"**

**Primary variant (`variant="primary"`):**
- Element: `<a href={VITE_HERMES_BOT_URL} target="_blank" rel="noopener noreferrer">`
- Background: `--mkt-accent` (`#7C3AED`)
- Text: `--mkt-accent-fg` (`#FFFFFF`), weight 600, `var(--mkt-font-sans)`, **Body** size (1rem)
- Border radius: `--mkt-radius-pill` (999px)
- Padding: `px-6 py-3` (24px / 12px) for standard size; `px-4 py-2` for `size="sm"` (nav instance)
- Box shadow: `0 0 24px color-mix(in oklch, var(--mkt-glow-from) 40%, transparent)` — violet glow halo
- Hover: background shifts to `--mkt-accent-hover` (`#6D28D9`), `transition-colors` 150ms ease-out
- Active: `scale(0.98)` transform, 50ms
- Icon: Telegram paper-plane icon from lucide-react (`Send` icon, 16px, `gap-2`) on left
- Min height: 44px (touch target)

**Ghost variant (`variant="ghost"`):** (reserved for future use in Phase 6 — not used currently)

**No-Telegram fallback (always rendered below the primary button):**
- Element: `<a href="mailto:${VITE_CONTACT_EMAIL ?? 'contact@lala.la'}">`
- Style: `text-[--mkt-muted-fg] underline underline-offset-4` — **Small** size (0.875rem), weight 400
- Always visible — not conditionally rendered based on Telegram detection (no reliable browser detection)

**Deep-link URL:** `import.meta.env.VITE_HERMES_BOT_URL` (build-time injection).
Assumed value: `https://t.me/DonnaFollowFun_bot?start=creator_onboard` — planner must verify
the exact `start` payload in `artifacts/hermes/src/` before committing the env var value.

**When `VITE_HERMES_BOT_URL` is empty string:** The primary button still renders with the
`href=""` (which opens no page) — acceptable because the contact fallback is always visible.
Alternatively, the planner may choose to hide the primary button and show only the fallback when
the env var is absent. Either approach satisfies MKT-09.

**Hero entrance animation (Phase 6 — minimal):**
One orchestrated Framer Motion `initial/animate` pass on page load — NOT scroll-triggered:
- Eyebrow (if rendered): `y: 8 → 0, opacity: 0 → 1`, delay 0ms, duration 250ms
- `<h1>` headline: `y: 16 → 0, opacity: 0 → 1`, delay 100ms, duration 400ms
- Subheadline: `y: 12 → 0, opacity: 0 → 1`, delay 200ms, duration 350ms
- CtaButton: `y: 8 → 0, opacity: 0 → 1`, delay 300ms, duration 300ms
- HeroOrb: `opacity: 0 → 1`, delay 400ms, duration 600ms

**CRITICAL:** The `<h1>` headline IS the LCP element. If `initial={{ opacity: 0 }}` is set without
`immediately-resolved SSR hydration`, LCP will regress. For a client-rendered SPA (no SSR), the LCP
is the first painted text. Acceptable approach: render `<h1>` at `opacity: 1` statically, and use
Framer Motion only on the outer wrapper (not the `<h1>` text node itself). Or: accept the LCP
trade-off and compensate in Phase 7. **Planner must choose one approach** — the contract permits
either but the executor must not animate the `<h1>` text to `opacity: 0` as an initial state without
a plan to recover LCP score.

**Scroll-reveal animations:** DEFERRED to Phase 7 (MKT-12). Phase 6 renders all sections at full
opacity on first paint. No `AnimatePresence` with scroll-triggered variants in Phase 6.

**Reduced-motion:** Apply `@media (prefers-reduced-motion: reduce)` to disable the hero entrance
stagger. Phase 7 enforces this across all scroll animations. In Phase 6, at minimum:
```css
@media (prefers-reduced-motion: reduce) {
```
