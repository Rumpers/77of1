<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fan browser → POST /api/twin/chat | untrusted input (message body, conversation_id cookie) |
| api-server → GMI Cloud | outbound HTTPS with bearer auth — server-side only |
| res.locals → handler body | trusted (set by middleware after verification) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Spoofing | HMAC conversation_id bypass | mitigate | verifyConversationId middleware (from 02-02) runs globally before routes; tampered token → 401; absent → new id issued |
| T-02-03-02 | Tampering | KYC gate bypass | mitigate | kycGate middleware enforces strict `=== "signed"`; handle required at route entry (existing 400 check); no `if (handle)` wrapper |
| T-02-03-03 | Info Disclosure | system prompt leak via prompt injection | accept | L2 meta-instruction in buildSystemPrompt mitigates; full L3 output-moderation lands in plan 02-05 (one wave later — acceptable since 02-03 ships unsafe-but-functional twin) |
| T-02-03-04 | Info Disclosure | raw fan PII to Helicone / GMI | mitigate | hashFanId used for Helicone fanId param; conversation_id (HMAC hex) used as session id — no email/IP/cookie value crosses boundary |
| T-02-03-05 | DoS | LLM cost amplification (unbounded /api/twin/chat) | accept | Phase 2 ships without rate limiting; mitigated by trial counter on web SPA (3 free turns) + future paywall; full rate limit deferred to Phase 3 hygiene |
| T-02-03-06 | Tampering | conversation_messages plaintext content | accept | D-03 — retentionCategory='transcript', 90-day TTL cleanup in Phase 4; conversation content is required for context, cannot be hashed |
| T-02-03-07 | Repudiation | LLM-generated reply blamed on creator | accept | disclosure_footer in every response + every Telegram message names twin as AI; D-02-12 enforces this |
| T-02-03-08 | Info Disclosure | logged fan PII | mitigate | Pino redacts authorization/cookie; no `logger.info({message: req.body.message})` allowed — code review gate in 02-03 task review |
| T-02-03-SC | Tampering | npm installs | accept | NO new package installs (i18next-http-middleware DROPPED per D-02-14; inline detectLocale used instead) |
</threat_model>
