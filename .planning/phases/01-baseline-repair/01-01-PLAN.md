<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer shell → Replit PG | `drizzle-kit push` executes DDL; any rogue schema edit lands directly in prod-shaped DB |
| api-server route → Replit PG | Once schema is live, every downstream Drizzle query inherits its constraints (NOT NULL, enums, FKs) |
| safety_audit_log write path → DB | Must store hashes only — schema enforces no raw-PII column exists |
| .env.example → developer shells / CI | Template drives `cp .env.example .env.local` workflows; stale SUPABASE_* entries encourage future drift back onto Supabase |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-00 | Tampering | .env.example listing dead SUPABASE_* keys long after migration | mitigate | Task 0 removes the three keys before any code migration begins (D-11 step 1); subsequent Wave 2 plans that still reference `process.env.SUPABASE_URL` will surface as live regressions |
| T-01-01 | Tampering | drizzle-kit push on legacy data | mitigate | Task 3 backfill checkpoint runs SQL UPDATEs to collapse old 8-state enum into 3-state before push; without this, push fails or silently corrupts (Pitfall #1) |
| T-01-02 | Information Disclosure | safety_audit_log schema accidentally accepts raw fan_id / message_text | mitigate | Task 1 omits raw-PII columns; Task 4 acceptance criterion explicitly asserts `information_schema.columns` has zero rows for `fan_id`/`message_text`/`content` on safety_audit_log (D-02, COMPLY-03) |
| T-01-03 | Elevation of Privilege | creator_kyc.status nullable → null bypass at gate | mitigate | Task 1 declares `status` with `.notNull().default("pending")`; Task 4 acceptance verifies `is_nullable=NO` and `column_default='pending'::kyc_status` (Pitfall #4) |

| T-01-05 | Spoofing | creators.handle collision allowing two creators on same /lala.la/[handle] | mitigate | Task 1 declares `handle text("handle").notNull().unique()` — DB-level uniqueness constraint |
| T-01-SC | Tampering | npm/pnpm installs | accept | No new packages added in Plan 01 (all Drizzle deps already present in catalog); legitimacy gate not applicable per RESEARCH.md "Package Legitimacy Audit" |
