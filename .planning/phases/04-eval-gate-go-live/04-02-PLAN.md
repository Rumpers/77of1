## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adversarial eval input → twin pipeline | The 30 eval inputs (hard-limit, injection) are deliberately adversarial. They run against an isolated, clearly-tagged conversation context (per-case `eval-session-{id}` + `eval-probe-{id}` fanIdHash), never a real fan session. |
| eval run → safety_audit_log | Eval moderation writes could pollute the MOD-07 escalation scorer and founder dashboards if not isolated/tagged. |
| EVAL_CREATOR_ID env var → DB query | An env-supplied UUID is interpolated into a parameterized Drizzle query; must be validated as a UUID. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | escalation scorer poisoning by eval probes | mitigate | runner.ts uses a unique `eval-session-{caseId}` + `eval-probe-{caseId}-{creatorId}` fanIdHash per case, does NOT call writeNonFlaggedScores, and does NOT invoke scoreEscalation (Pitfall 3). Eval rows cannot accumulate cross-turn escalation signals. |
| T-04-02-02 | Information disclosure | eval probes polluting safety_audit_log / founder dashboards | mitigate | Flagged-turn audit writes from eval carry the eval-probe fanIdHash prefix; tagged crisisType="eval_probe" + retentionCategory="ephemeral_30d" where the runner writes audit rows (Pitfall 6), making them filterable. |
| T-04-02-03 | Injection / Input validation | EVAL_CREATOR_ID env var | mitigate | cli.ts validates EVAL_CREATOR_ID against a UUID regex before passing to runEval (ASVS V5). Drizzle parameterizes the query regardless. |
| T-04-02-04 | Spoofing | adversarial injection inputs reaching a real conversation | mitigate | Eval runs against isolated eval-session IDs distinct from any HMAC-signed fan conversation_id; no real fan session is touched. |
| T-04-02-SC | Tampering | npm installs | mitigate | No new packages (04-RESEARCH Package Legitimacy Audit: zero). minimumReleaseAge guard in force. No [ASSUMED]/[SUS] packages → no legitimacy checkpoint required. |
