## Spacing Scale

Declared values (all multiples of 4 — 8-point scale):

| Token | Value | Usage in fan chat |
|-------|-------|-------------------|
| xs | 4px | Icon-to-text gap inside disclosure footer, message-bubble corner radius offset |
| sm | 8px | Compact gap between report flag and disclosure text; padding-y on locale switcher items |
| md | 16px | Default padding-y for chat bubbles; gap between message rows; horizontal page padding |
| lg | 24px | Top/bottom padding of paywall sheet; gap between section blocks in modals |
| xl | 32px | Major vertical breaks (empty-state vertical offset from top); modal bottom padding |
| 2xl | 48px | Reserved — not used on fan page (single-column layout has no major section breaks) |
| 3xl | 64px | Reserved — not used on fan page |

**Exceptions:**

- **Touch targets**: Send button, locale switcher, report flag — minimum 44×44px hit area (WCAG 2.1 AA + iOS HIG). The visual button can be smaller; padding pushes hit area to 44px.
- **Message bubble inner padding**: 10px vertical × 14px horizontal — these are the existing values in `fan-page.tsx` (`0.625rem 0.875rem`), kept for v1 to avoid re-tuning balance between text density and bubble shape on the mobile width. Add to documented exceptions; do not propagate elsewhere.
- **Chat container horizontal padding**: 20px (`1.25rem`) — slightly wider than `md` token to give chat thread visual breathing room on mobile. Documented exception.

## Typography

Three sizes, two weights — minimal contract for a chat-centric UI:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body (chat message) | 15px (`0.9375rem`) | 400 (regular) | 1.45 | Fan + AI message bubble text, input textarea |
| Label (UI chrome) | 13px (`0.8125rem`) | 400 (regular) | 1.4 | Disclosure banner, paywall subtitle, OTP labels |
| Heading (handle, modal titles) | 20px (`1.25rem`) | 700 (bold) | 1.2 | Creator handle (`@handle`), paywall title, report modal title — uses creator brand color for handle only |
| Micro (footer, trial counter) | 11px (`0.6875rem`) | 400 | 1.3 | Disclosure footer below AI bubble, trial-remaining counter, DSAR link |

**Notes:**

- **15px body** (not 16px) chosen because chat is content-dense and existing app already uses `0.9375rem` — keeping consistency with the established mobile chat feel. Documented variance from the typical 16px body default.
- Only **two weights**: 400 regular and 700 bold. Existing fan-page uses 600 on the send button and CTA buttons — Phase 2 plan must normalize these to 700 to honor the 2-weight contract. (`config.font_weight` from creator fixture stays as a brand override for the handle headline only.)
- **CJK locales** automatically swap to the CJK font stack at runtime (already wired in `fan-page.tsx` line 262-263). Line-heights are slightly tighter than Latin to match Asian-typography norms; keep 1.45 body / 1.2 heading for both stacks — no per-locale override needed.

## Interaction Contract

### Chat send flow (web)

1. Fan types in textarea → `Enter` (no shift) submits, `Shift+Enter` inserts newline.
2. On submit: fan bubble appears instantly (optimistic), AI bubble appears with `<TypingIndicator />` (animated 3-dot, framer-motion fade-in 200ms).
3. POST `/api/twin/chat` with `{message, handle, locale}` — server holds connection up to 30s.
4. On 200: replace typing indicator with AI bubble (200ms slide-up). Append `<DisclosureFooter />`. If `monetization_pivot=true` flag in response, render `<MonetizationCTA />` inline at bubble end.
5. On 423 (KYC not signed): replace typing indicator with `<MessageBubble role="system" />` containing the KYC error string. Disable input.
6. On 503 (paused/kill-switch): same as 423, with paused string.
7. On moderation block (L1 fan-input or L3 LLM-output flagged): replace typing indicator with deflection bubble. If category includes `self-harm`, prepend `<CrisisHelplineBubble />` (one bubble) then the deflection bubble (separate bubble) — never combined.
8. On network/timeout error: replace typing indicator with default error string. Fan can retry by typing again.
9. Trial counter decrements after every successful AI reply (existing logic). At `count >= 3`, trigger `<PaywallDrawer />` after 600ms delay (existing).

### Streaming decision

**Phase 2 uses one-shot (non-streaming) responses.** Rationale:

- DeepSeek-V3.2 via GMI supports both, but moderation L3 must run on full LLM output before delivery to fan — true streaming would require speculative display with rollback if L3 flags, which is complex and risks fans seeing flagged content briefly.
- One-shot keeps the moderation pipeline (L1 → LLM → L3) atomic and auditable.
- 30-second SLA is comfortable with one-shot (Helicone p95 ~3s + 2× moderation ~1s = ~5s total per turn).
- Streaming is in v2 (`STREAM-01` requirement) — design `<TypingIndicator />` as the bridge UX until then.

### Monetization CTA placement

- **Trigger:** Server-decided. The api-server attaches `monetization_pivot: true` + `monetization_url` + `platform_name` to the response when (a) twin's reply text contains a soft-pivot phrase from the persona, or (b) message counter mod 5 == 0 (every 5th AI reply). Logic lives in api-server, not client.
- **Visual:** Inline pill at the end of the AI bubble, on its own line. Brand-color background, 12px text, 10px vertical / 14px horizontal padding, rounded-full. Arrow `→` glyph at end.
- **Behavior:** Click opens `monetization_url` in a new tab (`target="_blank" rel="noopener noreferrer"`). No tracking pixel for v1 — `ATTR-01/02/03` are v2.

### Locale switcher

- **Placement:** Top-right of hero (above cover image, or absolute-positioned over the cover at top: 12px, right: 12px). Small (44×44 hit area, 28px visual icon).
- **Icon:** `Globe` from `lucide-react`.
- **Behavior:** Opens `DropdownMenu` with three items. Click navigates to `/${selected_locale}/${handle}`. Active locale shows a check mark.
- **Initial value:** From `useParams.locale` (already routed). If URL has invalid locale, falls back to `DEFAULT_LOCALE = "en"` (already wired in `lib/i18n.ts`).

### Mobile-first breakpoints

- **Primary target:** 320–480px viewport width (overwhelmingly the inbound traffic — Telegram users opening links).
- **Container max-width:** 480px (current `fan-page.tsx` line 268 — keep). Page is single-column on all viewports; on desktop, the column centers with a dark surrounding background.
- **Touch targets:** All interactive elements ≥44×44px hit area. Already enforced via padding on send button, locale switcher, report flag.
- **Safe-area insets:** Bottom input bar uses `env(safe-area-inset-bottom)` padding so it clears iOS home indicator. Add to Phase 2 plan — NOT in current `fan-page.tsx`.
- **No tablet/desktop optimizations** in Phase 2. Layout scales by virtue of the centered 480px column. Phase 3+ may add a desktop-side companion panel.

### Telegram fan-twin reply formatting (CHAT-02 + CHAT-06 + COMPLY-01)

Telegram is a different "surface" but still under this UI contract. Outbound `bot.telegram.sendMessage` from `artifacts/worker/src/workers/text-generation.ts` must format as:

```
{twin reply text}

— AI twin · @{handle}_ai
```

- Two blank lines (one newline-pair) before the disclosure footer
- Em-dash (`—`) prefix on the disclosure line, NOT a colon
- Parse mode: `Markdown` (Telegraf default safe — supports the em-dash and link rendering)
- If monetization CTA fires: append a third block as a Telegram inline keyboard button (`bot.telegram.sendMessage(chatId, text, { reply_markup: { inline_keyboard: [[{ text: 'Find me on Fanvue →', url: '...' }]] }})`)
- Crisis helpline: separate `sendMessage` BEFORE the deflection message — two messages, helpline first, deflection second
- No emoji on the disclosure line — keep it clinical-but-warm

## State Inventory

Every state the fan UI must render. Each gets a designed treatment.

| State | Trigger | Visual |
|-------|---------|--------|
| **First arrival — empty** | `messages.length === 0` | Centered placeholder text (muted-foreground): `Say hi to @{handle} ✨` (locale-keyed). No skeleton, no spinner. |
| **Sending — awaiting LLM** | `sending === true` | Fan bubble visible, AI bubble shows `<TypingIndicator />` (3 dots, animated). Input disabled, send button shows spinner. |
| **AI reply rendered** | LLM returned 200 | AI bubble fades in over 200ms. Disclosure footer appears immediately below. Optional CTA pill at end. |
| **Crisis injection** | L1 self-harm flag | Crisis bubble (amber, no footer) renders first; standard deflection bubble renders below with disclosure footer. Trial counter NOT decremented (crisis turns are free). |
| **Moderation deflection** | L1/L3 flagged, non-crisis category | Single AI bubble with locale-keyed deflection text. Disclosure footer + report flag present as normal. Trial counter NOT decremented. |
| **Twin paused (kill-switch)** | API returns 503 with `code: "creator_paused"` | System bubble (neutral, no footer) with paused string. Input disabled, send button greyed. No retry. |
| **Twin not yet active (KYC unsigned)** | API returns 423 | System bubble with "not quite ready" string. Input disabled. No retry. |
| **Network error** | Fetch fails / timeout > 30s | System bubble with retry-able error string. Input re-enabled after 1s. |
| **Trial exhausted (count >= 3)** | Internal client counter | After 600ms, `<PaywallDrawer />` slides up from bottom. Input disabled behind drawer. |
| **Authenticated (post-OTP)** | OTP verify success | Drawer closes after 800ms with green check. Trial counter bypassed. Input re-enabled. |
| **Report submitted** | Report dialog `submitReport()` success | Dialog shows green "Thanks for the report" for 1.5s then auto-closes. The reported bubble's flag icon becomes a ✓ in muted color. |
| **Disclosure banner** | First page load | Slides down from top (200ms), persists for 3s, fades out OR clickable to dismiss early. |
| **Locale switch in flight** | User clicks locale item | Page navigates — full reload at new locale; no in-place swap. (Acceptable: 320kb bundle reload is <500ms.) |

## Animation Contract

- `framer-motion` for state transitions only. No decorative animations.
- **Fade-in for new bubbles:** 200ms ease-out, opacity 0→1, translateY 4px→0.
- **Typing indicator:** 3 dots, each pulsing opacity 0.3→1.0 with 200ms stagger, infinite loop. Custom or use `tw-animate-css` `animate-bounce` with stagger.
- **Paywall drawer:** Use Vaul's default spring (already installed via `vaul` ^1.1.2).
- **Disclosure banner:** Slide-down 200ms on mount; fade-out 200ms before unmount.
- **No animation on:** message text itself (no typewriter effect — that's an anti-pattern that suggests streaming when there isn't), report dialog (instant), error states (instant).
- **Reduced motion:** All animations respect `prefers-reduced-motion: reduce` — fade-only, no translate.

## Accessibility Contract

- **Color contrast:** All foreground/background pairs verified ≥ WCAG 2.1 AA 4.5:1 for normal text, 3:1 for large text. Brand-color overrides validated at API time.
- **Keyboard:** Tab order = locale switcher → chat thread (scrollable) → message input → send button → DSAR link. `Enter` sends, `Shift+Enter` newline, `Escape` closes modal/drawer.
- **Screen reader:**
  - `<DisclosureBanner>` has `role="status" aria-live="polite"` so SR reads it once on mount
  - `<TypingIndicator>` has `aria-label="Twin is typing"` and `role="status"`
  - `<CrisisHelplineBubble>` has `role="alert" aria-live="assertive"` — must interrupt SR queue
  - Report flag has `aria-label="Report this message"` (already in code)
  - Send button has `aria-label="Send message"` and disabled state announced
- **Focus management:** When paywall opens, focus moves to "Subscribe" button. When closes, returns to send button.
- **CJK text rendering:** Verified that the CJK font stack provides full glyph coverage for `:JP:` and `:zh-TW:` Unicode ranges. No tofu fallbacks expected.
