| Boundary | Description |
|----------|-------------|
| adversarial eval input → twin pipeline | The 30 eval inputs (hard-limit, injection) are deliberately adversarial. They run against an isolated, clearly-tagged conversation context (per-case `eval-session-{id}` + `eval-probe-{id}` fanIdHash), never a real fan session. |

| EVAL_CREATOR_ID env var → DB query | An env-supplied UUID is interpolated into a parameterized Drizzle query; must be validated as a UUID. |
