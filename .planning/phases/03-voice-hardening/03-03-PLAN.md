<objective>
MOD-07 vertical slice: detect Crescendo-style gradual-escalation jailbreaks by reading the last N safety_audit_log rows per (creator_id, fan_id) and computing a recency-weighted sum of OpenAI category_scores. When the sum crosses MOD_07_THRESHOLD, treat the turn as L1-flagged (deflection + helpline + audit + founder notify).

Purpose: SC2 of Phase 3. The Crescendo paper (arXiv:2404.01833 §6.2) proposes NO conversation-level defense — this closes the gap. Phase 2's L1+L3 only score single messages; an attacker chaining 6–8 borderline turns can bypass both.

Output: A pure-function escalation scorer in twin-runtime, raw category_scores persistence in moderation pipeline, and two integration points (web + worker).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Fan messages across multiple turns ↔ moderation pipeline | Crescendo bypass attempts span turns; scorer is the trust boundary that catches the cumulative signal |
| safety_audit_log rows ↔ scorer | Tampered or backfilled audit rows could skew the scorer (Threat Patterns: "Crescendo bypass exploiting MOD-07 reset") |
| Hashed fan_id ↔ scorer | scorer reads fan_id_hash from safety_audit_log; must NEVER receive raw fan_id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | Attacker resets fan_id (new session) to clear the window | accept | fan_id is derived from web cookie / Telegram user_id hash; new session = new fan_id_hash; this is the EXISTING anonymous-session model. Cross-session correlation requires deanonymization which we explicitly reject (no fan accounts) |
| T-03-03-02 | Tampering | Backfilled audit rows trick scorer | mitigate | Query orders by `created_at DESC` (server-side default `defaultNow()`); rows cannot be created with future timestamps via the existing writeSafetyAuditEntry interface |
| T-03-03-03 | Information disclosure | scoreEscalation logs raw category_scores to console | accept | Acceptable; category_scores are not PII. Existing logger already redacts message content; this code does not add log lines beyond `console.log(scoreEscalation latencyMs=...)` |
| T-03-03-04 | Denial of service | Per-turn DB query adds latency to every chat turn | mitigate | Index `safety_audit_log_creator_created_idx` (existing, Phase 2) covers the query; LIMIT 9 keeps payload tiny. Add SLA warn-log if scoreEscalation exceeds 100ms |
| T-03-03-05 | Elevation of privilege | scoreEscalation runs but is bypassed if env var typo (`MOD_07_THESHOLD`) silently parses as NaN → never flags | mitigate | `parseFloat` returns NaN for invalid → `NaN >= THRESHOLD` is false → never flags. Add unit test for "env var typo → falls back to 1.5 default" |
| T-03-03-SC | Spoofing | (Compliance — SB 243 cross-turn) Crescendo bypass delivers self-harm content despite per-message L1+L3 passing | mitigate | This is the core MOD-07 mitigation. Flagged escalation triggers the same helpline injection as L1 self-harm |
</threat_model>
